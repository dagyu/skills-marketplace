## Guidelines

These are non-negotiable rules for all work in this repository:

1. **Test-driven.** No production code without a failing test that came first.
2. **YAGNI.** Build what is needed now, not what might be needed later.
3. **Small, reviewable changes.** One concern per task, one task per commit.
4. **Keep docs honest.** Update `docs/` and `README.md` in the same change that
   alters behaviour.

## Definition of done

A task is done when:

- [ ] The tests for it pass (and failed before the code existed).
- [ ] The developer has confirmed the implementation.
- [ ] Documentation and the README reflect the change.
- [ ] The work is committed.
